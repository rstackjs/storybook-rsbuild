## 快速开始

最快的上手方式是使用 Storybook CLI。

### 新项目

如果你正在搭建一个新项目，我们推荐先创建一个 Rsbuild 项目，然后再添加 Storybook。

```bash
# 1. Create a new Rsbuild project
npm create rsbuild@latest

# 2. Initialize Storybook (in the project root)
npx storybook@latest init
```

Storybook 会检测到 Rsbuild 并自动为你配置 `storybook-builder-rsbuild`，同时还会向你的项目添加示例 stories。若想了解生成的文件，请查看 [Storybook 项目结构](https://storybook.js.org/docs/configure/user-interface/sidebar-and-urls)。

### 已有项目

如果你正在将一个已有的 Storybook 项目迁移到 Rsbuild，请安装相应的 framework 包并更新你的 `.storybook/main.ts`。

详细的迁移指南请参阅 **[Frameworks](/guide/framework/react)** 章节。

## 使用

:::info Peer dependency 要求

- **`@rsbuild/core >= 1.0.1`** — Storybook Rsbuild 依赖稳定的 Rsbuild 1.0 运行时。背景信息请参阅 [1.0 发布公告](https://rsbuild.rs/community/releases/v1-0)。
- **`storybook >= 10.0.0`** — 这些包已针对 Storybook 10 验证。升级指引可参阅 [Storybook 发布说明](https://storybook.js.org/blog/storybook-10/)。

:::

使用 v8 兼容版本的包时，你无需直接安装 `storybook-builder-rsbuild`。`storybook-react-rsbuild`、`storybook-vue3-rsbuild` 等 framework 包已经为你内置了它。
